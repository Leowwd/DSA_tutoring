Lesson 5

Linked Lists, Stacks, Queues, and Deques

Arrays are built around indexing. Linked lists are built around references. Stacks, queues, and deques are restricted access patterns that appear everywhere: parsing, undo history, BFS, scheduling, simulations, and monotonic stack problems.

Example first: playlist editing

Imagine a music playlist. Users may add a song to the front, add a song to the end, remove the current first song, or move through songs one by one. An array can store the songs, but inserting at the front requires shifting every existing song.

A linked list takes a different approach. Each item stores a value and a reference to the next item. To add at the front, we create one new node and point it at the old first node. 

class Node {
    String song;
    Node next;

    Node(String song, Node next) {
        this.song = song;
        this.next = next;
    }
}

static Node addFirst(Node head, String song) {
    return new Node(song, head);
}

This operation is O(1). We do not move old nodes. We only allocate a new node and update one reference.

The reference model

A linked list is a chain of node objects. The list variable stores a reference to the first node, usually called head. 

head
 |
 v
["A"] -> ["B"] -> ["C"] -> null

The arrows are references. This is the main concept. Linked list bugs are usually reference-update bugs: forgetting to update head, losing the rest of the list, or creating a cycle by accident.

Drawing rule: before writing linked list code, draw the nodes and arrows. Then write the assignment statements that transform the old picture into the new picture.

Arrays vs linked lists

OperationArray / ArrayListSingly linked listReason

Linked lists are not "always faster." They make some updates cheap, but they give up fast random access and usually use more memory per element because every node stores references.

Interface first: linked list ADT

Start with operations. A simple linked list ADT may look like this: 

interface LinkedListADT<E> {
    void addFirst(E value);
    void addLast(E value);
    E removeFirst();
    E getFirst();
    int size();
    boolean isEmpty();
}

This interface intentionally does not include fast get(index), because singly linked lists are not good at random index access.

Singly linked list implementation

A practical singly linked list stores head, tail, and size. The tail reference makes addLast O(1). 

class SinglyLinkedList<E> implements LinkedListADT<E> {
    private static class Node<E> {
        E item;
        Node<E> next;

        Node(E item, Node<E> next) {
            this.item = item;
            this.next = next;
        }
    }

    private Node<E> head;
    private Node<E> tail;
    private int size;

    public int size() {
        return size;
    }

    public boolean isEmpty() {
        return size == 0;
    }

    public E getFirst() {
        if (head == null) throw new NoSuchElementException();
        return head.item;
    }
}

Implementing addFirst

Adding to the front has two cases: empty list and non-empty list. The code can handle both if we update tail only when the list was empty. 

public void addFirst(E value) {
    head = new Node<>(value, head);
    if (tail == null) {
        tail = head;
    }
    size++;
}
Before:
head -> ["B"] -> ["C"] -> null

After addFirst("A"):
head -> ["A"] -> ["B"] -> ["C"] -> null

Implementing addLast

If the list is empty, the new node becomes both head and tail. Otherwise, attach the new node after the old tail and then move the tail reference. 

public void addLast(E value) {
    Node<E> node = new Node<>(value, null);
    if (tail == null) {
        head = node;
    } else {
        tail.next = node;
    }
    tail = node;
    size++;
}

The order matters. If we set tail = node before connecting tail.next, we lose access to the old tail.

Implementing removeFirst

Removing the first node also has a special empty-list case and a special one-element case. 

public E removeFirst() {
    if (head == null) throw new NoSuchElementException();
    E answer = head.item;
    head = head.next;
    if (head == null) {
        tail = null;
    }
    size--;
    return answer;
}

When removing the only node, head becomes null. The list must also set tail to null; otherwise the tail would point to a node that is no longer in the list.

Reversing a linked list

Reversal is a classic linked list exercise because it forces careful reference management. 

static Node reverse(Node head) {
    Node prev = null;
    Node cur = head;
    while (cur != null) {
        Node next = cur.next;
        cur.next = prev;
        prev = cur;
        cur = next;
    }
    return prev;
}

State definitions:

  • prev: the reversed part of the list.
  • cur: the node currently being moved.
  • next: the rest of the original list, saved before changing cur.next.

Cycle detection: slow and fast pointers

A linked list can accidentally contain a cycle. Floyd's cycle detection uses two pointers: one moves one step, the other moves two steps. If there is a cycle, they eventually meet. 

static boolean hasCycle(Node head) {
    Node slow = head;
    Node fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}

Time is O(n), and auxiliary space is O(1). A HashSet solution is also possible, but it uses O(n) extra space.

Doubly linked lists

A doubly linked list stores both next and prev references. This makes it easier to remove a known node or move backward, but each node uses more memory. 

private static class DNode<E> {
    E item;
    DNode<E> prev;
    DNode<E> next;

    DNode(E item, DNode<E> prev, DNode<E> next) {
        this.item = item;
        this.prev = prev;
        this.next = next;
    }
}

Sentinel nodes

A sentinel node is a dummy node that does not store real data. It exists to simplify edge cases. For example, a circular doubly linked list can use one sentinel whose next points to the first real node and whose prev points to the last real node. 

class SentinelDeque<E> {
    private final DNode<E> sentinel;
    private int size;

    SentinelDeque() {
        sentinel = new DNode<>(null, null, null);
        sentinel.next = sentinel;
        sentinel.prev = sentinel;
    }
}

With a sentinel, the empty list is not represented by null. It is represented by sentinel.next == sentinel. This makes add/remove operations more uniform.

Stack ADT

A stack is LIFO: last in, first out. Think of plates stacked on a table. The most recently added item is the first one removed. 

interface StackADT<E> {
    void push(E value);
    E pop();
    E peek();
    int size();
    boolean isEmpty();
}

Stack applications include function calls, undo history, parsing parentheses, evaluating expressions, DFS, and monotonic stack problems.

Queue ADT

A queue is FIFO: first in, first out. Think of a line at a store. The earliest item added is the earliest item removed. 

interface QueueADT<E> {
    void enqueue(E value);
    E dequeue();
    E peek();
    int size();
    boolean isEmpty();
}

Queue applications include BFS, task scheduling, simulations, producer-consumer systems, and buffering.

Deque ADT

A deque, pronounced "deck", is a double-ended queue. It supports adding and removing at both ends. 

interface DequeADT<E> {
    void addFirst(E value);
    void addLast(E value);
    E removeFirst();
    E removeLast();
    E peekFirst();
    E peekLast();
    int size();
    boolean isEmpty();
}

A deque can behave like both a stack and a queue. In Java, ArrayDeque is usually the best default implementation.

Java collections: use ArrayDeque

Java has an old Stack class, but for modern Java code, prefer ArrayDeque. It is faster and cleaner for stack and queue operations. 

Deque<Integer> stack = new ArrayDeque<>();
stack.push(10);
stack.push(20);
int top = stack.pop(); // 20

Deque<Integer> queue = new ArrayDeque<>();
queue.addLast(10);
queue.addLast(20);
int first = queue.removeFirst(); // 10

Application 1: valid parentheses

Parentheses are naturally stack-shaped. Every closing bracket should match the most recent unmatched opening bracket. 

static boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '[' || c == '{') {
            stack.push(c);
        } else {
            if (stack.isEmpty()) return false;
            char open = stack.pop();
            if (c == ')' && open != '(') return false;
            if (c == ']' && open != '[') return false;
            if (c == '}' && open != '{') return false;
        }
    }
    return stack.isEmpty();
}

Time is O(n). Space is O(n) in the worst case, such as a string of all opening brackets.

Application 2: evaluate postfix expressions

In postfix notation, operators come after their operands. Example: 2 3 + 4 * means (2 + 3) * 4. 

static int evalPostfix(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();
    for (String token : tokens) {
        if (token.equals("+") || token.equals("-") ||
            token.equals("*") || token.equals("/")) {
            int b = stack.pop();
            int a = stack.pop();
            stack.push(apply(a, b, token));
        } else {
            stack.push(Integer.parseInt(token));
        }
    }
    return stack.pop();
}

static int apply(int a, int b, String op) {
    if (op.equals("+")) return a + b;
    if (op.equals("-")) return a - b;
    if (op.equals("*")) return a * b;
    return a / b;
}

Application 3: monotonic stack

A monotonic stack keeps values in increasing or decreasing order. It is useful for "next greater element" and temperature-style problems.

Example: for each day's temperature, how many days until a warmer temperature? 

static int[] dailyTemperatures(int[] temps) {
    int[] ans = new int[temps.length];
    Deque<Integer> stack = new ArrayDeque<>(); // indices, temps decreasing

    for (int i = 0; i < temps.length; i++) {
        while (!stack.isEmpty() && temps[i] > temps[stack.peek()]) {
            int prev = stack.pop();
            ans[prev] = i - prev;
        }
        stack.push(i);
    }
    return ans;
}

Each index is pushed once and popped once, so the time is O(n), even though there is a nested while loop.

Application 4: BFS uses a queue

Breadth-first search explores nodes by distance from the start. This requires a queue: first discovered, first explored. 

static void bfs(List<List<Integer>> graph, int start) {
    boolean[] seen = new boolean[graph.size()];
    Deque<Integer> queue = new ArrayDeque<>();
    seen[start] = true;
    queue.addLast(start);

    while (!queue.isEmpty()) {
        int node = queue.removeFirst();
        for (int next : graph.get(node)) {
            if (!seen[next]) {
                seen[next] = true;
                queue.addLast(next);
            }
        }
    }
}

We will study graphs later, but this example shows why queues are not just a small API detail. They encode the order of exploration.

Choosing the right structure

NeedUseReason
Fast index accessArrayList or arrayO(1) get by index
Add/remove at front and backDequeBoth ends are cheap

FIFO processingQueue behavior with ArrayDequeOldest item first
Teach references or pointer updatesLinked listNodes expose object-reference structure

Common mistakes

  • Using a linked list when random access is the main operation.
  • Forgetting to update tail when removing the last node.
  • Changing cur.next before saving the old next node during reversal.
  • Using Java's old Stack class instead of ArrayDeque.
  • Using removeFirst without checking whether the deque is empty.
  • Claiming a monotonic stack is O(n^2) because it has a nested loop, without counting pushes and pops.

Analysis routine

ANALYZE-LINEAR-STRUCTURE(problem):
    1. Identify the access pattern: index, front, back, LIFO, FIFO.
    2. Choose array, linked list, stack, queue, or deque.
    3. Define the state: head, tail, size, stack contents, or queue contents.
    4. For linked lists, draw the reference updates.
    5. Count each operation: add, remove, peek, search.
    6. State time and auxiliary space.
    7. State edge cases: empty list, one element, many elements.
