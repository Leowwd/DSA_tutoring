Lesson 15

Greedy Algorithms: Local Choices and Proofs

A greedy algorithm repeatedly makes the locally best choice and never goes back. Greedy is powerful when it is correct, but dangerous when the local choice does not lead to a global optimum.

Example first: interval scheduling

Given intervals with start and end times, choose the maximum number of non-overlapping intervals. The greedy rule is: always pick the interval that ends earliest. 

record Interval(int start, int end) { }

static int maxNonOverlapping(List<Interval> intervals) {
    intervals.sort(Comparator.comparingInt(Interval::end));

    int count = 0;
    int currentEnd = Integer.MIN_VALUE;
    for (Interval interval : intervals) {
        if (interval.start() >= currentEnd) {
            count++;
            currentEnd = interval.end();
        }
    }
    return count;
}

Why earliest end time? It leaves as much room as possible for future intervals. Choosing the shortest interval or earliest start time can fail.

When greedy is plausible

SignalWhat to askExample
Repeated local choiceCan I decide one item now and never regret it?Pick earliest-ending interval.
Sorting helpsDoes one order expose the safe choice?Sort by end time, deadline, or weight.
Priority changes over timeDo I need the current best item repeatedly?Use a heap for scheduling.
Cut or component choiceCan I safely add the cheapest edge crossing a boundary?MST algorithms.

Greedy vs dynamic programming

Greedy commits to one choice. DP keeps many possible subproblem answers. If you can prove the local choice is always part of some optimal solution, greedy may work. If choices interact in complicated ways, DP is safer.

QuestionGreedy answerDP answer
Do we keep alternatives?No.Yes, in states.
Proof styleExchange argument or stays-ahead.Induction on state recurrence.
Common bugLocal choice is not globally safe.State is incomplete or transition misses a case.

Proof pattern 1: exchange argument

An exchange argument says: take an optimal solution. If it does not use the greedy choice, replace part of it with the greedy choice without making the solution worse. Therefore, there exists an optimal solution that starts with the greedy choice. 

EXCHANGE-PROOF:
    Let G be the greedy first choice.
    Let OPT be an optimal solution.
    If OPT already contains G, we are fine.
    Otherwise, swap OPT's first conflicting choice with G.
    Show the new solution is still valid and no worse.
    Repeat on the remaining subproblem.

Proof pattern 2: stays-ahead

A stays-ahead proof compares the greedy solution to any other solution after each step. For interval scheduling, after choosing k intervals, greedy's kth end time is no later than any other valid schedule's kth end time. That means greedy always leaves at least as much room for the future.

Case study: meeting rooms

Given meeting intervals, find the minimum number of rooms required. Sort by start time and use a min-heap of current end times. The heap stores rooms currently in use. 

static int minMeetingRooms(int[][] intervals) {
    Arrays.sort(intervals, Comparator.comparingInt(a -> a[0]));
    PriorityQueue<Integer> ends = new PriorityQueue<>();

    int best = 0;
    for (int[] meeting : intervals) {
        while (!ends.isEmpty() && ends.peek() <= meeting[0]) {
            ends.remove();
        }
        ends.add(meeting[1]);
        best = Math.max(best, ends.size());
    }
    return best;
}

Time is O(n log n): sorting plus heap operations. Space is O(n) in the worst case.

Case study: jump game

Given nums[i] as the maximum jump length from index i, can we reach the end? Keep the farthest reachable index. 

static boolean canJump(int[] nums) {
    int farthest = 0;
    for (int i = 0; i < nums.length; i++) {
        if (i > farthest) return false;
        farthest = Math.max(farthest, i + nums[i]);
    }
    return true;
}

Invariant: after processing index i, farthest is the farthest position reachable using jumps from indices we could reach so far.

Greedy counterexample habit

Before trusting greedy, try to break it. For coin change, choosing the largest coin first works for many real coin systems, but not all systems. 

coins = [1, 3, 4]
amount = 6

Greedy: 4 + 1 + 1 = 3 coins
Optimal: 3 + 3 = 2 coins

This is why coin change is usually taught as DP unless the coin system has special structure.
