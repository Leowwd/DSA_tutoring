Lesson 9

HashMap, HashSet, and Hashing

Hashing is one of the most important space-time tradeoffs in DSA. By using extra memory, hash tables make membership, lookup, and counting operations expected O(1).

Example first: first repeated word

Given a list of words, return the first word that appears twice. The direct approach compares each word to all previous words. A hash set remembers what we have seen. 

static String firstRepeated(String[] words) {
    HashSet<String> seen = new HashSet<>();
    for (String word : words) {
        if (seen.contains(word)) return word;
        seen.add(word);
    }
    return null;
}

The set uses O(n) extra space. In exchange, each membership check is expected O(1), so the full algorithm is expected O(n).

Set and map ADTs

A set stores keys. A map stores key-value pairs. A set can be implemented as a map where only the keys matter. 

interface SimpleSet<E> {
    boolean add(E value);
    boolean contains(E value);
    boolean remove(E value);
    int size();
}

interface SimpleMap<K, V> {
    void put(K key, V value);
    V get(K key);
    boolean containsKey(K key);
    V remove(K key);
    int size();
}

The key operation is lookup by key. If order is not important, hash tables are often the best default.

Hash table idea

A hash table uses an array of buckets. A hash function converts a key into an integer. That integer is mapped to a bucket index. 

bucketIndex = hash(key) mod bucketCount

If the key is "utah", Java computes a hash code for the string. The table uses that hash code to decide which bucket should contain the key. 

buckets:
0: []
1: [("utah", 12)]
2: []
3: [("java", 7), ("data", 4)]
4: []

Collisions

A collision happens when two different keys map to the same bucket. Collisions are unavoidable because there are many possible keys but only a finite number of buckets.

Two common collision strategies are:

StrategyIdeaNotes
Separate chainingEach bucket stores a list of entriesEasy to understand and implement
Open addressingIf a bucket is occupied, probe for another bucketMore subtle deletion behavior

This lesson focuses on separate chaining because it makes the structure visible.

Entry and chained hash map

class Entry<K, V> {
    K key;
    V value;

    Entry(K key, V value) {
        this.key = key;
        this.value = value;
    }
}

A simple chained hash map can use an array of lists. 

class SeparateChainingMap<K, V> implements SimpleMap<K, V> {
    private ArrayList<Entry<K, V>>[] buckets;
    private int size;

    @SuppressWarnings("unchecked")
    SeparateChainingMap(int capacity) {
        buckets = new ArrayList[capacity];
        for (int i = 0; i < buckets.length; i++) {
            buckets[i] = new ArrayList<>();
        }
    }
}

Computing an index safely

Java hash codes can be negative. Use Math.floorMod to get a valid nonnegative bucket index. 

private int indexFor(K key) {
    return Math.floorMod(key.hashCode(), buckets.length);
}

Real Java HashMap also handles null keys and uses additional bit mixing. For learning, this version is enough.

Implementing put and get

public void put(K key, V value) {
    int index = indexFor(key);
    ArrayList<Entry<K, V>> bucket = buckets[index];

    for (Entry<K, V> entry : bucket) {
        if (entry.key.equals(key)) {
            entry.value = value;
            return;
        }
    }

    bucket.add(new Entry<>(key, value));
    size++;
}

public V get(K key) {
    int index = indexFor(key);
    for (Entry<K, V> entry : buckets[index]) {
        if (entry.key.equals(key)) {
            return entry.value;
        }
    }
    return null;
}

If the hash function spreads keys well and the table is not too full, each bucket is short, so operations are expected O(1).

Load factor and resizing

The load factor measures how full the table is: 

load factor = number of entries / number of buckets

If the load factor gets too high, buckets become crowded and operations slow down. The table resizes by creating more buckets and rehashing entries. 

private double loadFactor() {
    return (double) size / buckets.length;
}

public void put(K key, V value) {
    if (loadFactor() > 0.75) resize(buckets.length * 2);
    // then insert or update
}

Resizing is expensive because every entry must be placed into a new bucket array. But it happens occasionally, so hash table insertion is usually described as amortized expected O(1).

equals and hashCode contract

If an object is used as a key, Java needs two things:

  • equals: decides whether two keys are logically equal.
  • hashCode: decides which bucket the key belongs to.

Contract: if a.equals(b) is true, then a.hashCode() must equal b.hashCode(). The reverse is not required; unequal objects may have the same hash code. 

record Point(int row, int col) { }

Java records are convenient keys because they automatically implement value-based equals and hashCode.

If you write a normal class, override both methods together. 

class StudentId {
    private final String value;

    StudentId(String value) {
        this.value = value;
    }

    public boolean equals(Object other) {
        if (this == other) return true;
        if (!(other instanceof StudentId)) return false;
        StudentId that = (StudentId) other;
        return value.equals(that.value);
    }

    public int hashCode() {
        return value.hashCode();
    }
}

Use a set when you need membership, uniqueness, or duplicate detection.

HashMap in Java

HashMap<String, Integer> credits = new HashMap<>();
credits.put("Algorithms", 4);
credits.put("Linear Algebra", 4);

int algorithmCredits = credits.get("Algorithms");

Use a map when each key has an associated value: counts, indices, metadata, distances, parent pointers, or cached results.

Pattern 1: frequency map

Counting occurrences is one of the most common map patterns. 

static Map<Character, Integer> frequencies(String s) {
    Map<Character, Integer> freq = new HashMap<>();
    for (char c : s.toCharArray()) {
        freq.put(c, freq.getOrDefault(c, 0) + 1);
    }
    return freq;
}

Time is O(n). Space is O(k), where k is the number of distinct characters.

Pattern 2: two sum on an unsorted array

For each number x, ask whether we have already seen target - x. 

static int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> indexByValue = new HashMap<>();
    for (int i = 0; i < nums.length; i++) {
        int need = target - nums[i];
        if (indexByValue.containsKey(need)) {
            return new int[] {indexByValue.get(need), i};
        }
        indexByValue.put(nums[i], i);
    }
    return new int[] {-1, -1};
}

This turns a pair search from O(n^2) into expected O(n) time using O(n) space.

Pattern 3: group anagrams

Anagrams have the same letter counts. We need a key that represents the group. 

static List<List<String>> groupAnagrams(String[] words) {
    Map<String, List<String>> groups = new HashMap<>();

    for (String word : words) {
        int[] count = new int[26];
        for (char c : word.toCharArray()) {
            count[c - 'a']++;
        }
        String key = Arrays.toString(count);
        groups.computeIfAbsent(key, k -> new ArrayList<>()).add(word);
    }

    return new ArrayList<>(groups.values());
}

Sorting each word also works, but counting can be faster when the alphabet is fixed.

Pattern 4: prefix sum plus HashMap

Count subarrays with sum k. If the current prefix sum is S, we need a previous prefix sum S - k. 

static int countSubarraysWithSumK(int[] nums, int k) {
    Map<Integer, Integer> freq = new HashMap<>();
    freq.put(0, 1);

    int prefix = 0;
    int count = 0;
    for (int x : nums) {
        prefix += x;
        count += freq.getOrDefault(prefix - k, 0);
        freq.put(prefix, freq.getOrDefault(prefix, 0) + 1);
    }
    return count;
}

Pattern 5: longest consecutive sequence

Put all values in a set. Only start counting from numbers that do not have a predecessor. 

static int longestConsecutive(int[] nums) {
    Set<Integer> set = new HashSet<>();
    for (int x : nums) set.add(x);

    int best = 0;
    for (int x : set) {
        if (!set.contains(x - 1)) {
            int cur = x;
            int length = 1;
            while (set.contains(cur + 1)) {
                cur++;
                length++;
            }
            best = Math.max(best, length);
        }
    }
    return best;
}

Each number is part of one counted sequence, so expected time is O(n).

Pattern 6: caching and memoization

A map can remember expensive function results. Dynamic programming often uses this idea. 

static int fibMemo(int n, Map<Integer, Integer> memo) {
    if (n <= 1) return n;
    if (memo.containsKey(n)) return memo.get(n);
    int ans = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
    memo.put(n, ans);
    return ans;
}

The map changes exponential repeated recursion into linear work over the number of distinct states.

Complexity and assumptions

Hash table operations are usually expected O(1), but this depends on reasonable hashing and resizing. Worst case can be O(n) if many keys collide. In normal Java programming, HashMap and HashSet are reliable defaults, but complexity explanations should still say "expected."

OperationExpectedWorst case
putO(1) amortizedO(n)
getO(1)O(n)
containsKeyO(1)O(n)
removeO(1)O(n)

Common mistakes

  • Forgetting to update a frequency count after reading it.
  • Using get without checking whether the key exists.
  • Overriding equals but not hashCode.
  • Using a mutable object as a key and then changing fields that affect equality.
  • Saying HashMap operations are guaranteed O(1) instead of expected O(1).
  • Using a HashMap when sorted order or range queries are required.
